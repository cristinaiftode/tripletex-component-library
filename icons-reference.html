Tripletex Atlas — Icons & Illustrations Reference

50 most-used icons (out of 110 total), 19 brand-colored sidebar icons, and all 12 illustrations from the Atlas design system. Use this file to give AI prototyping tools real Tripletex SVG path data instead of guessed-from-training-data icons.

How to use:

1. Each icon card shows the kebab-case name (e.g. chevron-down) and the React component name (e.g. ChevronDownIcon).

2. View source on this file to copy the exact SVG path data — most icons use a 20×20 viewBox and fill="currentColor" so they inherit your CSS color. The brand sidebar icons use a 32×32 viewBox and a baked-in brand color.

3. Default icon colors should come from color: var(--text-primary). For status colors use the global palette: info #0A41FA, success #33AC2E, warning #F7C137, error #E83645.

4. Sidebar icons are special: the 19 icons in the Brand sidebar icons section below ship with their own brand colors (e.g. teal for Timeliste, red for Faktura). Use these — not generic monochrome icons — when prototyping the Tripletex left sidebar.

5. The remaining 60 icons are listed by name only in manifest.json → icons.names. If you need one not shown here, ask before inventing SVG path data.

Using illustrations in a prototype:

• When the repo is local: <img src="assets/illustrations/success.svg" alt="" />

• In a self-contained HTML prototype: use the GitHub raw URL — https://raw.githubusercontent.com/cristinaiftode/tripletex-component-library/main/assets/illustrations/<name>.svg

• Recommended sizes: 80px for inline accents, 120–160px for empty states, 200–240px for modal headers and onboarding pages.
